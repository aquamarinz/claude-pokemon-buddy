# Claude Pokémon Buddy — 设计规格 (Design Spec)

> 状态：Design v1.2 · **codex R2 通过（修 §12 v2 玩家开关矛盾后）· brainstorm 收尾 → writing-plans**。日期：2026-05-30。
> 配套：调研 `docs/research/2026-05-30-pokemon-mechanics-adaptation.md`（机制数值/来源）· 视觉 `mockups/01–07`（版面定稿见 06 的 V2、心情见 07、天气见 05 的中文-A）。

## 1. 概述
送给一位**硬核宝可梦玩家**朋友的桌面硬件礼物。一块**单色 1-bit 反射屏**小设备，USB-C 接朋友的 Windows 电脑常驻桌面：**左半显示他的 Claude 用量仪表盘**，**右半是一只用他的 Claude 用量"养"大的伊布(Eevee) buddy**——会进化、有性格、断更会蔫会退化（可恢复），并埋满硬核情怀彩蛋。

灵感源自用户自有项目 `~/Projects/codexbar-dashboard`（Mac 副屏用量 kiosk）；本项目 = 其"宝可梦化 + 硬件化 + Windows 版"。

## 2. 目标 / 非目标
**目标**
- 真实、稳定的 Claude 用量 ambient 显示（朋友"瞄一眼"即知额度/花费/天气/室温）。
- 一只情感上"活着"的 buddy：**可养任意宝可梦**（数据驱动进化引擎 + PokéAPI），用量驱动养成、持久个性；伊布为旗舰款（8 向分支最炫设备能力）。
- **难度固定 Normal（开局锁定、不可改）**：有真实代价但永远可挽回（断更→亲密回落/buddy 蔫，长期→可逆退化，streak 有护盾；绝不永久失去/真离家）。**不给玩家难度/平衡/彩蛋开关——那会拆掉游戏。** dashboard = **训练师伴侣图鉴 App（只读为主）**。
- 大量"硬核玩家一看一听就会心一笑"的彩蛋，且**单色屏+喇叭原生还原**初代 GB 质感。
- **可重启幸存**：host 重启 / 设备断电重插 → 养成进度不丢、自动恢复。
- 送礼友好：朋友侧零维护，gift-giver(用户)预先配置好。

**非目标（明确不做）**
- 不做永久死亡（离家最多"可找回"）。
- 不照搬 Tamagotchi 的日常喂养 chore（用量=被动喂养）。
- v1 不做 WiFi（走 USB 串口）、不做计步（无加速度计）。**选初始宝可梦 + 在已拥有的之间切换 = v1**；**收藏成长（孵蛋/解锁更多）= v2**。
- 不碰假梗/猎奇：见 §8.3。

## 3. 硬件（事实，详见调研/官方）
Waveshare **ESP32-S3-RLCD-4.2**：4.2" 反射 LCD，原生 300×400，驱动 **ST7305 单色黑白 1-bit**（非彩色非灰阶彩），~119ppi，无背光/无触摸，**横用 → 400×300**。ESP32-S3-WROOM-1-N16R8（240MHz/16MB Flash/8MB PSRAM/WiFi+BLE）；**喇叭(ES8311 codec + ES7210)+双麦**；**SHTC3 温湿度**；**PCF85063 RTC**；microSD；18650；**KEY/BOOT 两实体键**；USB-C(供电+数据)。

## 4. 架构
**总原则——单一渲染权威**：所有像素由 **host(Windows) 渲染成 400×300 单色图**，经 USB 串口推给 ESP32；**ESP32 只做"笨显示器"**（收图刷屏 / 回传按键 / 上报传感器 / 放音）。唯一在设备本地发生的是音频播放（host 只发"播第 N 号声音"）。理由：把复杂度放在用户擅长侧(JS/HTML)，固件极简、最不易错；避免分布式渲染不一致。

```
朋友 Windows PC (Node 单进程, 开机自启)                 ESP32-S3-RLCD-4.2
┌──────────────────────────────────────────┐   USB    ┌────────────────────────┐
│ usage(ccusage) ─┐                          │  串口    │ 固件(thin):              │
│ weather(Open-Meteo)─┤→ pet(养成 sim) → render│ <────>  │ · 收帧→刷 ST7305 屏      │
│ sensors(收ESP32上报)─┘    ↓ state.json(持久) │         │ · 读 KEY/BOOT→回传事件   │
│                        transport(串口收发)   │         │ · 读 SHTC3→上报          │
│                        audio(发"播声"指令)   │         │ · 收"播声ID"→喇叭        │
└──────────────────────────────────────────┘         │ · 音频 flash/PSRAM    │
                                                       └────────────────────────┘
```

**Host 模块**
- `usage`：周期(~60s)跑 ccusage（读朋友 `~/.claude` 日志，朋友需装 Claude Code）→ 归一化 {5h窗口%, 周%, today$, todayTokens, resets, 按类型分流的tokens}。**fail-closed 解析**（参考 codexbar `snapshot.sh`：schema drift → 不覆盖旧值，只标失败）。
- `weather`：Open-Meteo（按经纬度，新西兰）→ {WMO状况码→中文, temp, 体感, 最高/最低, 湿度, 风, 降水概率, 日出日落}；空气质量走其免费 Air-Quality 端点（可选）。失败保留 last-known。
- `sensors`：接收 ESP32 上报的 SHTC3 室内温湿度。
- `pet`：养成模拟核心（§7），每 tick 用 usage+RTC+weather+temp 推进 pet 状态。
- `render`：合成 400×300 1-bit 帧（§6/§11）。
- `transport`：串口自动认 COM（按 ESP32 USB VID/PID，Espressif 原生 0x303A 或桥接芯片 VID）；推帧（变化时/周期，RLE 压缩）；收按键/传感器事件。
- `state`：pet 状态原子写 JSON，reboot 幸存（§9）。
- `audio`：发"播声 ID"指令。
- `web`：**本地控制 dashboard**（localhost 网页，host 进程内服务）——见 §4.1。
- `autostart`：Windows 开机自启（注册表 Run 键 / 任务计划 / 托盘小程序，plan 时定）。

**ECU(ESP32) 固件**：串口帧收发 + ST7305 驱动(处理 300×400↔400×300 旋转) + KEY/BOOT(短/长/双击)上报 + SHTC3 周期上报 + 音频播放(UI 音预存 flash + 叫声 SOUND_LOAD 进 PSRAM，§4)。v1 不联网。

> **取数真实性说明**：ccusage 的 cost 是按公开定价**估算**（非真实账单）、5h 窗口是从本地日志**建模**的 burn（非 Anthropic 官方剩余%）。对 ambient 仪表盘 + 养成驱动**足够**；streak 由 host 从每日用量历史推导并持久化。

**素材策略（v1 = 本地种子包，不冷启动依赖外网）**：精灵/进化链/叫声由 host **本地缓存**，ESP32 只收渲染好的帧。v1 随附**本地种子包**（伊布全家族 + setup 选的初始 box），**不**在 v1 拉全 Pokédex；"按需拉 PokéAPI 全量并缓存" = v1.5。版权：copyrighted 精灵/叫声/BGM **不入 repo、不再分发**——runtime 取用、限个人礼物用途（§17）。

**音频（v1 单一模型，收敛之前三方案）**：① UI/系统音(升级/治疗/拔群/低电量) **预存 ESP32 flash**，host 发 `PLAY(id)` 触发；② 宝可梦**叫声**在**切换 buddy 时**由 host 经 `SOUND_LOAD` 把当前 buddy 叫声一次性传给 ESP32 缓存(PSRAM)，之后同样 `PLAY(id)` 触发。**v1 不做实时 PCM 流**（与帧带宽争用）。

### 4.1 本地控制 Dashboard（玩法切换台）
host 进程内开一个 **localhost 网页 = 训练师伴侣图鉴 App**（设备 2 键才是日常玩的地方；dashboard **以只读为主**：欣赏 / 追踪 / 管理 buddy，不编辑规则）。**游戏公司原则：不暴露难度 / 平衡 / 彩蛋开关（暴露=拆游戏 + 剧透彩蛋）。** 内容：
- **MY BUDDY（只读）**：当前形态 / 等级 / 亲密度 / 此刻心情 / 性格·特性·天赋(IV)·签名(Characteristic)·徽章 + 下一进化进度条。
- **BOX 收藏**：已拥有缩略图，**切换当前出战**（选显示哪只，非改数值），图鉴完成度。
- **JOURNEY 旅程（只读）**：里程碑时间线 + 近 7 日"训练量"(用量)曲线。
- **SECRETS 秘密陈列馆（只读）**：彩蛋**亲手发现后**才解锁陈列（不剧透 / 无开关）。
- **SETTINGS（仅真·偏好）**：buddy 取名、免打扰时段、音量、天气定位、设备配对状态；难度显示为 `NORMAL·锁定`（只读）。
- **LIVE 预览**：400×300 设备屏只读镜像。

> dashboard 视觉 = **"Pokédex Console"** 现代 web app（图鉴红 / GB 绿 / 镜头灯 + 像素(Press Start 2P)/VT323 字 + 内嵌单色预览），与设备单色屏成对比。已实现 `demo/dashboard.html`（含 MY BUDDY 只读图鉴页、BOX 切换、JOURNEY 时间线+训练量曲线、SECRETS 发现-陈列馆、SETTINGS 纯偏好）。

## 5. 屏幕设计（版面定稿 = mockup 06 的 V2）
**Layout A**：左栏 Claude 仪表盘 / 右栏伊布 buddy。Game Boy / Pokédex 像素风；mockup 用墨 `#14160d` 于纸 `#c9ccbf`，真机纯黑白。

**左栏三段（上→下）**
1. **CLAUDE 用量**（主角，放大）：`72%` 大号(64px) + `5H WINDOW` + `↺ reset` → `WEEK 41%` 条 + `↺ wk reset` → `today $/tok`。
2. **天气**（中文-A）：`多云 19° 体感17°` / `最高22° 最低14° · 东南风3级 · 降水30%`。
3. **里外温湿度**（钉底部）：`室内 23.4°·56%`(SHTC3) / `室外 19.0°·64%`(天气 API)。

**右栏 buddy 区**：伊布精灵(灰度抖动) + 心情叠加层(§7.5) + 气泡 + **状态行 `Lv.7 ♥♥♥♡ 🔥6d`** + 细 EXP 条。进化/彩蛋时整屏特效。

**字体**：等宽/点阵；中文用像素 CJK 字体（如 Zpix）保持 GB 味。汉字真机 ≥14px 才清晰。

## 6. 数据→画面映射（左栏字段表）
| 字段 | 来源 | 说明 |
|---|---|---|
| 5H% / reset | ccusage(blocks) | 主 hero 数 |
| WEEK% / reset | ccusage | 周窗口 |
| today $/tok | ccusage | 估算 |
| streak | host 推导 | 连续使用天数 |
| 天气(状况/温/体感/高低/风/降水/日出落) | Open-Meteo | 中文 |
| 室内温湿度 | SHTC3(ESP32 上报) | — |
| 室外温湿度 | Open-Meteo | — |
| 时钟 | host/RTC | — |

**百分比语义（必修：否则不可验）**：
- **5H%** = 当前活跃 5 小时 block 的 token 用量 ÷ **参考额度**。参考额度 = setup 配置的计划 token 预算（首选）／否则取近 N 个 block 的**滚动高水位**作默认。**本地建模值(modelled)，非 Anthropic 官方剩余额度** → UI 加小标 `LOCAL`/`est.`。
- **WEEK%** = 近 7 天 token ÷ 周参考额度（同逻辑）。
- 数据源 = **ccusage JSON**（`blocks`/`daily`/`session`），**pin 版本、只吃 JSON、schema drift 即 fail-closed**（不解析表格文本）。

## 7. 宝可梦 Buddy 系统
**口径**：可养**任意宝可梦**（数据驱动进化引擎 + PokéAPI 全精灵 / 进化链）。首次开机**选初始宝可梦**（御三家时刻彩蛋）；**KEY 切换当前展示的 buddy**——在你已拥有 / 养过的之间切，各自独立养成状态（这就是你最初要的"可切换"，与养成不冲突）。**伊布是旗舰款**（8 向分支吃满设备真实世界映射）；其余宝可梦用更简单触发各成"用法人格"。**v1 = Eevee-first**：随附**固定初始 box**（伊布 + setup 时 gift-giver 选的几只御三家，本地种子包），可在这固定 box 间 **KEY 切换**；**进化引擎数据驱动、架构支持所有宝可梦，但 v1 只 seed 伊布家族 + 初始 box 的进化数据**——"拉全 Pokédex / 孵蛋解锁更多" = v1.5/v2。（解决 §15 "已拥有来源不明"矛盾：v1 的 box 是固定预置，非运行时解锁。）

### 7.1 双轨养成
- **等级 Level（吃用量，永久不掉）**：每日 EXP = f(当日 tokens)，带**每日软上限**（防一天爆肝秒满 + 防脚本刷量）。累计 → `Lv.N`。用量**按类型分流**为 EV 式专精——**注意：ccusage 不天然知道"代码/长文/高频"语义，只能从项目/模型/会话长度/token 节奏做启发式(heuristic)，未知一律归 balanced**。
- **亲密度 Bond（吃坚持，会衰减）**：0–255，进化阈值 **160**（现代值）；只要当日用过 Claude → +亲密 & streak+1。软上限 ~180，冲 255 需"特殊喂养"(里程碑/树果类事件)。
- 两轨解耦：等级=累计(永久)、亲密=坚持(可衰减)，对应玩家"用得多" vs "来得勤"。

### 7.2 进化引擎（数据驱动 · 支持所有宝可梦 · 伊布为旗舰示例）
引擎**读 PokéAPI 进化链 + 一张"原作触发方法 → 现实数据轴"映射策略表**，对**任意宝可梦**生效——**加一只宝可梦 = 加数据，不写代码**。通用映射：升级→用量/等级阈值；友好度→亲密度(满160)；时段→RTC；石头→手动选；天气→真实天气；能力值(如 Tyrogue→三 Hitmon)→用量类型(EV)；地点(苔藓/冰岩)→室温/天气；**通信交换进化**(单机不可)→替代触发(里程碑 / "交换道具"手动解锁)。
**伊布旗舰示例**（详见调研 §1）：亲密满 160 后伊布"发光"，**下次互动时**进化（把昼/夜时机控制权交玩家）：

| 进化 | 触发(现实映射) | 人格 |
|---|---|---|
| 太阳伊布 Espeon | 亲密满 + **白天(RTC)** 触发 | 日间生产型 |
| 月亮伊布 Umbreon | 亲密满 + **夜晚(RTC)** 触发 | 夜猫子型 |
| 仙子伊布 Sylveon | 亲密满 + **高互动**(摸摸/对话计数达标) | 陪伴宠爱型 |
| 叶伊布 Leafeon | 亲密满 + **室温暖+湿度高/晴**(SHTC3+天气) | 温润型 |
| 冰伊布 Glaceon | 亲密满 + **室温冷/真实下雪**(SHTC3+天气) | 冬日型 |
| 水/雷/火伊布 | 累计用量解锁后**手动按键选石** | 氪强度型 |

**多条件冲突解析（必修）**：亲密满 160 → 伊布"发光"进入**可进化态**；进化**不自动**，等玩家按 KEY 触发。触发瞬间算**当前合格分支集**（RTC 昼/夜、care 计数、SHTC3 室温/湿度、真实天气）：
- 合格分支**唯一** → 直接进化。
- **多个合格**（如白天+暖湿同时满足 Espeon 与 Leafeon）→ **屏上列候选、玩家 KEY 选**（保留"亲手决定"仪式感）。
- 三石(水/雷/火) = **随时手动覆盖**，无视环境/亲密分支。
- 玩家不选时的默认优先级：care(仙子) > 昼夜(太阳/月亮) > 环境(叶/冰)。

进化彩蛋：1-bit 黑白反相闪烁 + 方波进化音 + `恭喜进化!`。

### 7.3 个性层（几乎零像素成本，终身不变）
- **Nature 性格**：领养/前 7 天画像定，终身。绑用量风格(急性子/慢性子…) + 口味喜恶(喂"喜欢的味"涨亲密多)。
- **IV 天赋**：6 项随机(0–31)，终身 → 抽卡稀缺感 + "天赋鉴定"。
- **Characteristic 签名**：最高 IV 决定的一句(如"爱睡午觉")，终身。
- buddy 摘要页(KEY 长按)显示：`性格 / 天赋 / 签名 / 等级 / 徽章`。

### 7.4 难度（固定 Normal · 开局锁定 · 玩家不可改）
**只 ship 一个作者化调好的难度 = Normal**，无难度选择器（游戏公司原则：调好的体验 > 让玩家自拼；且"不可重新调的礼物默认就不该是惩罚性硬核"）。**Normal = 有真实代价但永远可挽回**：断更→亲密回落 + buddy 蔫/失落 → 长期→**可逆退化**(降一阶，回归数日复原) → streak 有**护盾**抵一次断更；**绝不永久失去、绝不真离家/死**。每档恶化**屏上可见预警**，不背刺。下面机制（原创层，标注非原作）按此"可挽回"基调调：
- 断更**弃养升级阶梯**：漏1天→streak 硬清零(先扣护盾)+亲密−+sad；2–3天→亲密加速掉+喊饿；4–6天→**生病**(骷髅,需回来主动"喂药/玩游戏"才愈)；7+天病着→**退化一阶**(雷丘式降级，**可逆**)；14+天→**离家出走**(留空窝+字条，**可找回**：持续回归+照料数天哄回)。
- **streak 护盾**：长 streak 攒 1 个护盾，抵一次断更。
- **公平铁律**：每档恶化**屏上肉眼可见 + 喇叭升级预警**，给足挽救窗口；**不静默背刺**（这是"硬核"和"垃圾设计"的分界）。退化态招式"报恩→迁怒"作闹脾气彩蛋。
- 已得等级/进化**永不永久失去**（退化可逆、离家可找回、不死）。

### 7.5 心情态（7 种，叠加层+姿势，与具体宝可梦解耦，见 mockup 07）
happy(5h0–50%) / focused(50–80%) / strained 冒汗(80–99%) / fainted 躺倒(100%限额) / celebrate 弹跳(窗口reset) / shocked 闪电(今日花费飙升) / sleep 变淡Zzz(深夜或长断更)。由当前用量+RTC 实时推导，**叠加层 host 合成到精灵上**。
**动效**：每态有 idle/动作（待机轻浮 · happy 弹跳 · strained 发抖+汗滴 · fainted **90° 躺平**+起伏 · celebrate 跳跃+星 · shocked 抖动+电火花 · sleep 呼吸+Zzz 上飘）；真机 = **低帧率(2–4fps)推帧**，非 60fps。已在 `demo/index.html` 实现并经用户调校。

### 7.6 真实世界 → 游戏机制 1:1（卖点）
- **RTC 昼夜**(套 HGSS 边界) → 待机动画/BGM 切 + 太阳/月亮伊布分叉。
- **真实天气 → Pokémon GO 式属性加成光环**(映射表见调研 §5) + 雨天进化。
- **室温(SHTC3)** → 叶/冰伊布环境门控 + 冷热情绪。
- **日历月 → 第五世代四季**(背景换皮/小鹿形态；**南半球新西兰月份翻转**)。

### 7.7 互动 + 按键映射

## 9. 持久化与可重启幸存
- `state.json` 含：`schemaVersion`、pet 全状态(species/stage、level、EXP、bond、streak、shield、Nature、IV[6]、Characteristic、age、neglect/sickness、evolution、彩蛋flags)、`lastTickAt`、`lastDailySettlementDate`、`discoveredSecrets[]`、box 列表。
- **原子写**：写 tmp → `fsync` → rename（Windows 覆盖语义用 `ReplaceFile`/重试）；保留**上一份备份**；读时 JSON 校验失败 → 回退备份 → 再失败则**安全重建**(保等级/进化、重置易失态)并告警（主+备份双坏时，等级/进化来源 = 周期 checkpoint 快照；plan 落 salvage 策略）。
- **离线天数幂等结算（必修）**：日奖励/断更惩罚以 `lastDailySettlementDate` 为锚，**按缺失日补算且幂等**——host 多次重启 / PC 跨日休眠**绝不重复加 bond、不重复扣惩罚**；超长缺口**封顶**(最多按 N 天算)。
- 设备断电/重插/休眠唤醒：host 自动重连 COM、整屏全刷。
- DoD：`host 重启 ×3` + `设备拔插` + `PC 睡一夜跨日` 后，进度一致、无重复结算。

## 10. Host↔ESP32 链路契约（v1 必须锁定）
- **物理层**：ESP32-S3 **原生 USB CDC**（**非 115200 UART！**）——全速 USB，实测吞吐 ~0.5–1MB/s+。15KB 满帧 ~15–30ms，2–4fps 余量充足。（demo footer 的 `@115200` 是占位错误，作废。）
- **帧格式**：`magic | type | seq | len | payload | crc32`。
- **显示帧**：默认推**脏矩形 + RLE**（仪表盘几乎静止、只 buddy 区动 → 每帧通常 <1–2KB）；整屏全刷仅在切页/进化/重连时。host 维护上一帧算 diff。
- **Host→ESP32**：`FRAME`(脏区 1bpp+RLE) / `SOUND_LOAD`(id,bytes) / `PLAY`(id) / `CONFIG`。
- **ESP32→Host**：`HELLO`(固件版本+板型+VID) / `BUTTON`(key,短/长/双击) / `SENSOR`(temp,humidity) / `ACK`(seq) / `NACK`(seq) / 心跳。
- **流控**：每帧带 seq + ESP32 `ACK`；**未 ACK 不发下一帧**(stop-and-wait，单 buddy 帧足够)；超时重发 N 次后标 stale。背压时**丢中间帧只发最新**(ambient 不补帧)。
- **失联/重连**：拔插或休眠唤醒 → host 重扫 COM(VID/PID) → 重握手 → 整屏全刷一帧。

## 11. 渲染管线
- host 渲 400×300 **灰度** → **抖动(Bayer 有序抖动为主，稳定无帧间噪点闪烁；精灵可 Floyd–Steinberg)** → 打包 1bpp → 脏区+RLE 推(§10)。
- **屏驱动锁定**：ST7305，**1-bit / 2 级**（Waveshare 官方页确认，非 4 级灰阶）。**以 Waveshare 官方示例的 ST7305 初始化序列为准**，不靠实现期试错。300×400 原生 ↔ 400×300 旋转在固件侧处理。
- **引擎抉择 → 定 B（为"礼物机零维护"）**：**B) 纯 Node canvas（`@napi-rs/canvas`/skia）**——无浏览器依赖、启动快、无崩溃重启/字体打包/Edge 路径问题，最适合朋友 Windows 机长期跑。代价=放弃直接复用 mockup HTML，需用 canvas 重绘版面(版面已定稿，量可控)。**A) headless Chromium** 降为备选；若用须解决 Edge 路径/字体打包/离线 sprite/截图耗时/崩溃自恢复——对零维护礼物是负担。

## 13. 成功标准（可验证）
1. 设备 USB 接入 + host 启动 → N 秒内显示仪表盘+buddy。
2. 仪表盘数字与 **ccusage JSON + §6 公式一致**（modelled，UI 标 `LOCAL/est.`）；天气/室温为真实值。
3. 用量累积 → 等级/EXP 增；当日有用量 → 亲密+streak+1。
4. 亲密满 160 + 白天触发 → Espeon；夜晚 → Umbreon；室温冷触发 → Glaceon（各分支按条件正确）。
5. 断更 → 依阶梯衰减/生病/退化，且**每档有屏上可见预警**；回归后可恢复；护盾正确抵一次断更。
6. `host 重启 ×3` + `设备拔插` + `PC 跨日休眠` → state 恢复、进度一致、**无重复结算**。
7. 至少一个签名彩蛋(闪光 或 大木开场白)按条件触发并正确呈现。
8. 连续运行 ≥ 数日无崩溃/无进度损坏。

## 17. 实现契约补遗（codex review R1 响应）
- **维护后门（非玩家开关 · gift-giver 隐藏入口）**：`reset`(重开档)、`repair`(修坏档)、**`pause`/vacation 模式**(暂停断更惩罚——解"难度锁定+朋友放假"隐患)、`export`(state 备份)。不破坏游戏性，不出现在玩家面板。
- **断更边界规则**：base-stage(伊布/未进化)**不可再退化**；当前 buddy 离家时**自动切到 box 内其他**；回归恢复动作(连续 N 天有用量 / 一次主动照料)明确化。
- **天气韧性**：Open-Meteo 非 SLA → 指数 backoff、last-known+TTL 缓存、定位错/无网时**降级显示**(灰条/"--")，不阻塞 buddy。
- **本地 web 安全**：dashboard server **bind 127.0.0.1**；输入(取名/定位)持久化前校验；防被网页诱导请求(CSRF)。
- **Windows 运维（零维护礼物必备）**：打包(单 exe/服务)、开机自启、托盘、**日志轮转**、**崩溃自恢复(supervisor)**、USB 断连/PC 休眠唤醒后自愈。
- **平衡参数表（plan 必产出）**：EXP/亲密/streak护盾/各档衰减/进化阈值的**具体公式 + 起步数值表 + 每日结算时机(本地午夜，幂等)**——design 给基调，plan 落可模拟的 v0 参数。
- **demo ≠ 产品**：`demo/index.html` 的"调性/心情覆盖/彩蛋触发"控件是**模拟沙盒**，**不可照搬**进产品 UI(产品无这些开关)。
- **素材版权**：宝可梦精灵/叫声/BGM 不入 repo、不再分发；runtime 取用、限个人礼物用途。
